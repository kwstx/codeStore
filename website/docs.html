Documentation

Everything you need to get started with Engram, your local-first code memory companion.

Installation

Engram is distributed as a VS Code extension. To install:

  1. Open Visual Studio Code.
  2. Go to the Extensions view (Ctrl+Shift+X or Cmd+Shift+X).
  3. Search for "Engram".
  4. Click Install.

First Run

After installing, Engram activates automatically when you open VS Code. It runs in the background and starts tracking your coding patterns immediately—no configuration needed.

All data is stored locally in your workspace storage. Nothing is uploaded to external servers.

Core Features

  • Mistake Detection: Automatically fingerprints errors and warnings you encounter. When you repeat a mistake, Engram warns you and shows how you fixed it before.
  • Paste Tracking: Detects when you paste code and tracks snippet reuse. Get notified when you've used the same snippet multiple times.
  • Security Scanner: Real-time scanning for dangerous patterns like hardcoded secrets, eval usage, and SQL injection risks. Warnings appear inline as you paste code.
  • Prompt Timeline: View your prompt history in the sidebar. Track which prompts led to successful or failed sessions.
  • Memory Cards: Hover over warnings to see contextual memory cards showing when you last encountered this issue and how you fixed it.

Sensitivity Control

Click the shield icon in the status bar to toggle between two sensitivity modes:

  • Breeze (Quiet): Only warns when you've made the same mistake 3+ times. Shows only critical security issues.
  • Strict (Audible): Warns after 2+ occurrences of a mistake. Shows all security warnings.

Commands

Access Engram commands via the Command Palette (Ctrl+Shift+P or Cmd+Shift+P):

  • Engram: Search Snippets - Browse and reuse previously pasted code snippets
  • Engram: Log Prompt - Manually log a prompt and check for similarity to past failures
  • Engram: Mark Failure - Mark the current session as failed for future reference
  • Engram: Exclude this File - Exclude the current file from Engram tracking

Storage Location

All Engram data is stored locally in your VS Code workspace storage:

  • Mistakes: .vscode/engram/mistakes/
  • Snippets: .vscode/engram/snippets/
  • Sessions: .vscode/engram/vibe_sessions/
  • Security Exemptions: .vscode/engram/exemptions/

You can safely delete these folders to reset Engram's memory for a workspace.
